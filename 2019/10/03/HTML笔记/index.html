发表于 2019-10-03 更新于 2019-10-13

超文本标记语言 (Hyper Text Markup Language)

  • 基本格式

    <html> 与 </html> 之间的文本描述网页。

    定义了整个 HTML 文档。
    <body> 与</body> 之间的文本是可见的页面内容。

    定义了 HTML 文档的主体

  • 元素

    1 标题

    <h1> - <h6> 等标签进行定义的

    <h1>This is a heading</h1>

2 段落

通过<p> 标签定义的

<p>This is a paragraph.</p>

3 水平线

<hr /> 元素可用于分隔内容

4 注释

<!– This is a comment –>

5 折行
<br />换行
6 引用
<q> 用于短的引用

<q>构建人与自然和谐相处的世界。</q>

构建人与自然和谐相处的世界。

浏览器通常会在q 元素周围包围引号。

<blockquote>用于长引用
浏览器通常会对 blockquote 元素进行缩进处理。

7 缩写
<abbr>定义缩写或首字母缩略语

WHO

<abbr title=”World Health Organization”>\WHO</abbr>

对缩写进行标记能够为浏览器、翻译系统以及搜索引擎提供有用的信息。

<acronym>一个首字母缩写

WWW

<acronym title=”World Wide Web”>WWW</acronym>

<dfn>

  1. 如果设置了 <dfn> 元素的 title 属性，则定义项目：

    <p><dfn title=”World Health Organization”>WHO</dfn> 成立于 1948 年。</p>

    eg:

    WHO 成立于 1948 年。

  2. 如果 <dfn> 元素包含具有标题的 <abbr> 元素，则 title 定义项目：

    <p><dfn><abbr title=”World Health Organization”>WHO</abbr></dfn> 成立于 1948 年。</p>

    WHO 成立于 1948 年。

  3. 否则，<dfn> 文本内容即是项目，并且父元素包含定义。

    <p><dfn>WHO</dfn> World Health Organization 成立 于 1948 年。</p>

    WHO World Health Organization 成立 于 1948 年。

    8 联系信息
<address>

用于联系信息（定义文档或文章的联系信息<作者/拥有者>）

用于联系信息（定义文档或文章的联系信息<作者 拥有者>）

​ 通常以斜体显示。大多数浏览器会在此元素前后添加折行。

9 著作的标题
<cite>定义著作的标题

<p><cite>The Scream<cite> by Edward Munch. Painted in 1893.</p>

The Scream by Edward Munch. Painted in 1893.

浏览器通常用斜体显示<cite>元素

10 双向重写
<bdo>双向重写

<bdo dir=”rtl”>This text will be written from right to left</bdo>

This text will be written from right to left

定义双流向覆盖（bi-directional override）。

11 图像
<img>图像

<img src=”w3school.jpg” width=”104” height=”142” />
可调宽高
<img> 是空标签，并且没有闭合标签。
要在页面上显示图像，你需要使用源属性（src）。src 指 “source”。源属性的值是图像的 URL 地址（指存储图像的位置）。
插入动画图像的语法与插入普通图像的语法没有区别

图像背景（无需style=”background）

<body background=”/i/eg_background.jpg”>
gif 和 jpg 文件均可用作 HTML 背景。如果图像小于页面，图像会进行重复。

排列图像
bottom 对齐方式是默认的对齐方式
图像 <img src =”/i/eg_cute.gif” align=”middle”> 在文本中
图像 <img src =”/i/eg_cute.gif” align=”top”> 在文本中
带有图像的一个段落。图像的 align 属性设置为 “left/right”。图像将浮动到文本的左/右侧。
<img src =”/i/eg_cute.gif” align =”left”>

12 alt替换文本属性

无法载入图像时，替换文本属性告诉读者她们失去的信息
如果您把鼠标指针移动到图像上，大多数浏览器会显示 “alt” 文本
仅支持文本的浏览器无法显示图像，仅仅能够显示在图像的 “alt” 属性中指定的文本

不需要可变的字母尺寸，以及可变的字母间距）支持固定的字母尺寸和间距

和换行。

<h4>两行三列：</h4>

17 超链接（链接）

<a>链接 <a href=”http://www.w3school.com.cn">This is a link</a>
点击This is a link可进入链接

有两种使用 <a> 标签创建链接的方式：
“链接文本” 不必一定是文本。图片或其他 HTML 元素都可以成为链接
图像<a href=”/example/html/lastpage.html”>
<img border=”0” src=”/i/eg_buttonnext.gif” />
</a>
1.通过使用 href 属性（规定链接的目标） - 创建指向另一个文档的链接 开始标签和结束标签之间的文字被作为超级链接来显示
2.通过使用 name 属性 - 创建文档内的书签
可以使用 name 属性创建 HTML 页面中的书签（书签不会以任何特殊方式显示，它对读者是不可见的）。
<a name=”tips”>基本的注意事项 - 有用的提示</a>
当使用命名锚（named anchors）时，我们可以创建直接跳至该命名锚（比如页面中某个小节）的链接
<a name=”label”>锚（显示在页面上的文本）</a>

没有下划线的链接

<head>
<meta http-equiv=”Content-Type” content=”text/html; charset=gb2312” />
<meta http-equiv=”Content-Language” content=”zh-cn” />
</head>

<body>

<a href=”/example/html/lastpage.html” style=”text-decoration:none”>
这是一个链接！
</a>

</body>
</html>

target 属性（可以定义被链接的文档在何处显示）
设置为 “_blank”会在新窗口打开文档：
<a href=”http://www.w3school.com.cn/" target=”_blank”>Visit W3School!</a>
<a href=”#C4”>查看 Chapter 4。</a> <h2><a name=”C4”>Chapter 4</a></h2>

导航框架

导航框架包含一个将第二个框架作为目标的链接列表。名为 “contents.htm” 的文件包含三个链接

跳转至框架内的一个指定的节

link.htm”文件内指定的节使用 <a name=”C10”> 进行标识。

iframe

<iframe src=”URL”></iframe> URL 指向隔离页面的位置。

某些老式的浏览器不支持内联框架。如果不支持，则 iframe 是不可见的。
设置高度和宽度<iframe src=”demo_iframe.htm” width=”200” height=”200”></iframe>

删除边框

frameborder 属性规定是否显示 iframe 周围的边框。
设置属性值为 “0” 就可以移除边框：

<iframe src=”demo_iframe.htm” frameborder=”0”></iframe>

用作链接的目标（target）

链接的 target 属性必须引用 iframe 的 name 属性

<iframe src=”demo_iframe.htm” name=”iframe_a”></iframe>
<p><a href=”http://www.w3school.com.cn" target=”iframe_a”>W3School.com.cn</a></p>

绝对文件路径
相对路径

字符实体

不间断空格（non-breaking space）
增加空格的数量，您需要使用 &nbsp; 字符实体。

统一资源定位器（URL)

用于定位万维网上的文档（或其他数据）

scheme://host.domain:port/path/filename

scheme - 定义因特网服务的类型。最常见的类型是 http
host - 定义域主机（http 的默认主机是 www）
domain - 定义因特网域名，比如 w3school.com.cn
:port - 定义主机上的端口号（http 的默认端口号是 80）
path - 定义服务器上的路径（如果省略，则文档必须位于网站的根目录中）。
filename - 定义文档/资源的名称

URL 字符编码

会将字符转换为可通过因特网传输的格式
只能使用 ASCII 字符集来通过因特网进行发送
“%” 其后跟随两位的十六进制数来替换非 ASCII 字符
不能包含空格。URL 编码通常使用 + 来替换空格。

<!DOCTYPE>声明

帮助浏览器正确地显示网页
它为浏览器提供一项信息（声明），即 HTML 是用什么版本编写的
